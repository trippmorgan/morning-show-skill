# PlayoutONE Database Schema Reference
*Updated: 2026-03-30 — revised after morning show incident*

Database: `PlayoutONE_Standard`  
Server: `localhost\p1sqlexpress`  
Auth: Windows integrated (`-E` flag with sqlcmd, or `-Username PLAYOUTONE_APPS -Password PlayoutONE.` with Invoke-Sqlcmd)

---

## Architecture — How PlayoutONE Schedules Audio

```
Audio table          → master library (all tracks, UIDs, file paths, markers)
DPL files            → schedule input (generated by Music1 or by us)
AutoImporter         → watches F:\PlayoutONE\Import\Music Logs\ and imports DPLs
Playlists table      → internal LOG (what played and what will play — managed by PlayoutONE)
```

**The Playlists table is a LOG, not a queue.** PlayoutONE owns it. We must use the AutoImporter to schedule content — never write to Playlists directly.

## Audio Table

Stores every audio item in the library. One row per file.

### Key Columns

| Column | Type | Nullable | Notes |
|--------|------|----------|-------|
| **UID** | nvarchar | NOT NULL | Unique identifier. Also the filename (`{UID}.mp3`) |
| **Title** | nvarchar | | Track title |
| **Artist** | nvarchar | | Artist name |
| **Filename** | nvarchar | | Filename only (e.g. `90005.mp3`) — not full path |
| **Length** | int | NOT NULL | Duration in **milliseconds** |
| **TrimIn** | int | NOT NULL | Start trim in ms. Set to `0` |
| **TrimOut** | int | NOT NULL | End trim in ms. **Must equal Length** — NEVER 0 |
| **Extro** | int | NOT NULL | Crossfade point in ms. **Must equal Length - 3000** — NEVER 0 |
| **Intro** | int | NOT NULL | Intro point in ms. Set to `0` |
| **Type** | int | NOT NULL | `16` = music/content |
| **Category** | int | NOT NULL | Category ID. Use `0` for show content |
| **Deleted** | bit | NOT NULL | `0` = active, `1` = hidden from playout |
| **AutoDJ** | bit | NOT NULL | `0` = don't include in AutoFill rotation |

**Total NOT NULL columns: 42.** Use the UPSERT pattern below — do not try to INSERT manually.

### ⚠️ Critical Marker Rule

| Marker | Wrong | Correct | Effect of Wrong Value |
|--------|-------|---------|----------------------|
| TrimOut | `0` | `= Length` | PlayoutONE thinks track is 0ms — instant skip |
| Extro | `0` | `= Length - 3000` | PlayoutONE fires end-of-track at 0ms — crash loop |

**These were the root cause of the 2026-03-30 incident. Never leave them at 0.**

### UPSERT Pattern (safe for show content)

```sql
SET QUOTED_IDENTIFIER ON;

DECLARE @uid NVARCHAR(10) = '90005';
DECLARE @len INT = 892000;  -- actual length in ms
DECLARE @extro INT = 889000; -- len - 3000

IF EXISTS (SELECT 1 FROM Audio WHERE UID = @uid)
BEGIN
    UPDATE Audio SET
        Title = 'Morning Show Hour 1',
        Artist = 'Dr Johnny Fever',
        Filename = '90005.mp3',
        Length = @len,
        TrimIn = 0, TrimOut = @len, Extro = @extro, Intro = 0,
        Type = 16, Category = 0, Deleted = 0, AutoDJ = 0
    WHERE UID = @uid;
END
ELSE
BEGIN
    -- Use publish.sh which handles all 42 NOT NULL columns correctly
    -- Do not attempt manual INSERT
    RAISERROR ('Use publish.sh for new Audio inserts', 16, 1);
END
```

## Playlists Table

PlayoutONE's internal schedule log. **Do not write to this directly.**

### Key Columns

| Column | Type | Notes |
|--------|------|-------|
| **GIndex** | decimal | **Primary key.** Format: `YYYYMMDDHH.NNNN`. Unique per row. |
| **Name** | varchar(50) | DPL file name: `YYYYMMDDHH.dpl` |
| **AirTime** | varchar(10) | Scheduled air time: `HH:MM:SS` |
| **Order** | real | Position within hour |
| **UID** | varchar(10) | References Audio.UID |
| **Title** | varchar(600) | Track title (copied from Audio on import) |
| **Artist** | varchar(255) | Artist (copied from Audio on import) |
| **SourceFile** | varchar(255) | **Full path to audio file.** AutoImporter sets this. Must not be blank. |
| **Length** | float | Duration in ms |
| **Len** | float | Effective length (after trim) |
| **LenOriginal** | float | Original length |
| **Chain** | bit | `1` = auto-chain to next track |
| **Type** | int | `0`=marker, `16`=music, `17`=station ID, `26`=commercial |
| **Played** | bit | `1` = has been played |
| **Done** | bit | `1` = finished playing |
| **MissingAudio** | bit | `1` = file not found on disk |
| **ActualAirTime** | datetime | When it actually aired |

### Type Values — What to Touch

| Type | Meaning | Rule |
|------|---------|------|
| `0` | START/SOFTMARKER | **NEVER modify** — structural markers |
| `16` | Music / content | Safe to read |
| `17` | Station ID / liner | **NEVER modify** |
| `26` | Commercial | **NEVER modify** |

## DPL File Format

Music1 hourly schedule files. AutoImporter reads these and populates Playlists.

**Filename:** `YYYYMMDDHH.dpl` (e.g. `2026040605.dpl` = April 6, 2026, 5 AM)  
**Format:** Tab-delimited, 14 columns  
**Drop path:** `F:\PlayoutONE\Import\Music Logs\` (**not** `C:\PlayoutONE\data\playlists\`)

### Column Reference

| Col | Field | Value for Show Content |
|-----|-------|----------------------|
| 1 | UID | e.g. `90005` |
| 2 | Chain | `-1` (auto-chain) |
| 3 | Extro | `-1` (use Audio table value) |
| 4 | Original Extro | `-1` |
| 5 | Fade | `-2` (use media finder setting) |
| 6 | Command | *(empty)* |
| 7 | Oversweep | `-2` |
| 8 | Scheduler Recon ID | `0` |
| 9 | Split | `-2` |
| 10 | ISCI | *(empty)* |
| 11 | Notes | *(empty)* |
| 12 | Airtime | *(empty)* |
| 13 | MergePoint Name | *(empty)* |
| 14 | Segue Envelopes | *(empty)* |

### Example DPL Row (tab-separated)
```
90005	-1	-1	-1	-2		-2	0	-2
```

## AutoImporter

Watches `F:\PlayoutONE\Import\Music Logs\` continuously.

When a new `.dpl` file appears:
1. Reads each UID from the file
2. Looks up UID in Audio table — gets Title, Artist, Length, SourceFile
3. Inserts row(s) into Playlists for that hour
4. Sets SourceFile = full path to audio file (`F:\PlayoutONE\Audio\{UID}.mp3`)
5. Removes DPL from import folder

**This is why Audio table markers must be correct BEFORE the DPL is dropped.**

AutoImporter processes files within ~10–15 seconds of drop.

## HTTP API (Port 81)

Available commands for show management:

| Command | Response | Notes |
|---------|----------|-------|
| `GET CURRENT PLAYER NOW_PLAYING` | `Player\tTitle\tArtist` | Works on Standard |
| `GET PLAYER 1 STATUS` | `STATUS,1,PLAYING` or `STATUS,1,EMPTY` | Works |
| `PLAY NEXT` | *(empty on success)* | Works — advances queue |
| `AUTOMATION ON` | `-255` | **Standard Edition — does not work** |
| `PLAY UID [n]` | `-255` | **Standard Edition — does not work** |
| `LOAD PLAYLIST [name]` | `-255` | **Standard Edition — does not work** |

**All write/control commands except PLAY NEXT return -255 on Standard Edition.**

## Common Queries

```sql
-- Confirm show is scheduled
SELECT Name, UID, Title, SourceFile, Length, MissingAudio, Played, Done
FROM Playlists
WHERE Name LIKE '20260406%' AND UID LIKE '9000%'
ORDER BY Name, [Order];

-- Check Audio markers for show UIDs
SELECT UID, Title, Length, TrimIn, TrimOut, Extro, Deleted
FROM Audio
WHERE UID IN ('90005','90006','90007','90008');

-- What's playing right now
SELECT TOP 1 Name, UID, Title, Artist, ActualAirTime
FROM Playlists WHERE Played=1 ORDER BY ActualAirTime DESC;

-- What's queued next
SELECT TOP 5 Name, [Order], UID, Title, Artist
FROM Playlists WHERE Played=0 AND Done=0 AND Type=16
ORDER BY Name, [Order];
```

## AutoImporter — Detailed Behavior (2026-03-30 test findings)

### First-Import-Wins
AutoImporter does **not** overwrite existing Playlists entries for a DPL name. If `2026040605.dpl` already has rows in Playlists, dropping a new `2026040605.dpl` will be ignored.

**Correct sequence:**
1. `DELETE FROM Playlists WHERE Name='YYYYMMDDHH.dpl' AND Type=16`
2. `Remove-Item F:\PlayoutONE\Import\Music Logs\Imported\YYYYMMDDHH.dpl`
3. Drop new DPL into `F:\PlayoutONE\Import\Music Logs\`
4. AutoImporter creates fresh entries ✅

### AutoImporter Does NOT Set Audio Markers
AutoImporter copies UID, Title, Artist, Filename into Playlists. It does **not** populate TrimOut or Extro from the Audio table into Playlists rows. Those values must be set in the **Audio table** before the DPL is dropped — AutoImporter reads them at import time.

### Audio.Filename Must Match Disk Exactly
`Audio.Filename` must be the exact filename on disk (e.g. `90005.mp3`). If the file on disk is `MORNING-SHOW-H5.mp3` but `Audio.Filename = '90005.mp3'`, PlayoutONE silently skips — no error, no alert.

Always verify:
```sql
SELECT UID, Filename FROM Audio WHERE UID='90005';
-- Filename must match what's actually in F:\PlayoutONE\Audio\
```

### Music1 Timing Conflict
Music1 runs daily (3 AM + on-boot via Windows Scheduled Task) and can also run on-demand. When Music1 regenerates DPLs and they're imported, they overwrite any existing Playlists entries for those hours.

**Publish sequence must account for this:**
- Check Music1 last activity before dropping our DPLs
- If Music1 ran recently, wait 60 seconds and re-check
- Our DPL must be the LAST one imported for those hours
