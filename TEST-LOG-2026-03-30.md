### Critical Discovery: AutoImporter Does NOT Set Audio Markers

After importing both DPLs, the Audio table markers were unchanged:
- 90001: TrimOut=3592000, Extro=3589000 (manually set earlier ✅)
- 99901: TrimOut=0, Extro=0 (still zero! ❌)

**AutoImporter only creates Playlists entries from the DPL data. It does NOT analyze audio files or update the Audio table.**

→ Manually fixed 99901 markers: TrimOut=599980, Extro=594980

**Implication:** Segmentation provides NO advantage for automatic marker analysis. We must always set markers manually via SQL regardless of file size.

### SourceFile in Playlists

AutoImporter sets `Playlists.SourceFile` to the DPL file path (e.g., `\\P1-WPFQ-SRVS\PlayoutONE\Import\Music Logs\2026033010.dpl`), NOT the audio file path. This is normal — all Music1-imported entries show the same pattern. PlayoutONE uses `Audio.Filename` + the configured audio directory to locate files.
